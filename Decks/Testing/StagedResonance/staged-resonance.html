Test deck for presentationMode: "staged" — four activity variants, one per slide.

Background

How Staged Mode Works

Adding "presentationMode": "staged" to the Resonance payload changes how the activity presents its question set to students.

Standard mode (default)

  • All questions appear in the activity panel at once.
  • Multiple-choice answer choices are visible from the moment the activity opens.
  • Students can respond to any question in any order, and all timers start together.

Staged mode

  • One question is shown at a time; the teacher controls when each question advances.
  • Multiple-choice questions display the stem only first; the teacher reveals the answer choices when ready, and the response timer starts at that moment.
  • Free-response questions show the full prompt immediately, since there is nothing to hide before the timer starts.
Test A

Free Response (Staged)

A single free-response question with a 60-second response window.

Expected behavior

  • The activity launches automatically on slide entry.
  • One free-response prompt appears in the activity panel; students type a response and a 60-second countdown runs.
  • Staged mode has no visible effect for a single FR question: behavior is identical to standard mode.

Payload

JSON
{
  "presentationMode": "staged",
  "questions": [
    {
      "id": "qa1",
      "type": "free-response",
      "text": "In one sentence...",
      "order": 0,
      "responseTimeLimitMs": 60000
    }
  ]
}
Test B

MC — Stem First

A single multiple-choice question in staged mode. This is the primary scenario for staged behavior: the question stem appears before the answer choices are shown.

Expected behavior

  • The activity launches; the question stem is visible but the four answer choices are hidden.
  • The teacher clicks "Reveal Choices" (or equivalent) to display the options and start the 30-second response timer.
  • Correct answer: final.

Payload

JSON
{
  "presentationMode": "staged",
  "questions": [{
    "id": "qb1",
    "type": "multiple-choice",
    "text": "Which keyword...",
    "order": 0,
    "responseTimeLimitMs": 30000,
    "options": [
      { "id": "qb1a", "isCorrect": false },
      { "id": "qb1b", "isCorrect": true  },
      { "id": "qb1c", "isCorrect": false },
      { "id": "qb1d", "isCorrect": false }
    ]
  }]
}
Test C

Three-Question Staged Set

A mixed set in staged mode: one graded MC question, one free-response question, and one opinion poll with no correct answer.

Expected behavior

  • Q1 (MC, 30 s): stem shows; teacher reveals choices. Correct answer is 4 (the number of rows).
  • Q2 (FR, 45 s): free-response prompt appears immediately; students type a response.
  • Q3 (Poll, 20 s): three choices, no isCorrect: true entry, so the activity runs in single-select poll mode.
  • The teacher advances each question manually; questions do not auto-advance.

Question set (abbreviated display)

JSON
{
  "presentationMode": "staged",
  "questions": [
    {
      "id": "qc1",
      "type": "multiple-choice",
      "text": "arr.length for new int[4][7]?",
      "order": 0,  "responseTimeLimitMs": 30000,
      // options: 7, 28, 4 (correct), 11
    },
    {
      "id": "qc2",
      "type": "free-response",
      "text": "Real-world 2D array use case?",
      "order": 1,  "responseTimeLimitMs": 45000
    },
    {
      "id": "qc3",
      "type": "multiple-choice",
      "text": "Confidence check...",
      "order": 2,  "responseTimeLimitMs": 20000,
      // 3 options, none isCorrect (poll mode)
    }
  ]
}

Test E

Inline Code in Stem and Choices

A single MC question in staged mode where the stem contains two inline code spans and every answer choice is itself a code expression wrapped in backticks.

Expected behavior

  • In the stem-only phase, the two inline code spans (matrix and new int[3][4]) should render with code styling, not raw backticks.
  • After the teacher reveals choices, all four options should render as formatted code expressions, not as backtick-delimited strings.
  • Correct answer: matrix[0][3] (index 3 is the last of 4 columns).

Payload

JSON
{
  "presentationMode": "staged",
  "questions": [{
    "id": "qe1",
    "type": "multiple-choice",
    "text": "...last element of `matrix`,
      declared as `new int[3][4]`?",
    "order": 0,
    "responseTimeLimitMs": 30000,
    "options": [
      { "text": "`matrix[0][3]`", "isCorrect": true  },
      { "text": "`matrix[0][4]`", "isCorrect": false },
      { "text": "`matrix[0].last()`", "isCorrect": false },
      { "text": "`matrix[1][3]`", "isCorrect": false }
    ]
  }]
}
Test F

Fenced Code Block in Stem

A single MC question in staged mode where the stem includes a fenced ```java code block embedded via \n escapes in the JSON string. Answer choices are plain integers.

Expected behavior

  • In the stem-only phase, the three-line Java snippet should render as a syntax-highlighted (or at minimum monospace) code block, not as raw backtick fence markers.
  • After choices are revealed, the four plain-text number options should render normally with no code styling.
  • Correct answer: 15 (10 + 5).

Payload

JSON
{
  "presentationMode": "staged",
  "questions": [{
    "id": "qf1",
    "type": "multiple-choice",
    "text": "What does this snippet print?\n\n
```java\nint x = 10;\nx += 5;\nSystem.out.println(x);\n```",
    "order": 0,
    "responseTimeLimitMs": 30000,
    "options": [
      { "text": "10", "isCorrect": false },
      { "text": "15", "isCorrect": true  },
      { "text": "5",  "isCorrect": false },
      { "text": "50", "isCorrect": false }
    ]
  }]
}
Test G

Code in Stem and Choices

A two-question staged set that combines every markdown code scenario: a fenced code block in a free-response stem (Q1), and a fenced block plus inline code choices in a multiple-choice question (Q2).

Expected behavior

  • Q1 (FR): the 7-line Java snippet in the stem should render as a code block; students type a prose description in response.
  • Q2 (MC, staged): the fenced code block in the stem renders during the stem-only phase; after choices are revealed, the four inline-code options (`3`, `5`, etc.) should render as code, not raw backtick strings.
  • Correct answer: 5 (arr[1][0] is 3; arr[0][1] is 2; 3 + 2 = 5).

Question set (abbreviated display)

{
  "presentationMode": "staged",
  "questions": [
    {

```",

```java\nint[][] arr={{1,2},{3,4}};\narr[1][0]+=arr[0][1];\n```",
